如何创建线程？有几种方式？

  1. 继承 Thread 类：重写 run() 方法，然后实例化并调用 start() 方法启动。
  2. 实现 Runnable 接口：实现 run() 方法，然后将 Runnable 实例作为参数传递给 Thread 的构造函数，再调用 start() 方法。（更推荐，因为Java是单继承，实现接口更灵活）
  3. 实现 Callable 接口：与 Runnable 类似，但 call() 方法可以有返回值，并且可以抛出异常。通常配合 线程池获取线程结果集，最后阻塞等待结果。
  4. 使用线程池（ExecutorFramework）：通过 Executors 工具类创建线程池，提交任务（Runnable 或 Callable）给线程池执行。这是目前最主流和推荐的方式。

线程死锁

死锁产生条件

互斥条件：一个资源每次只能被一个线程使用。
请求与保持条件：一个线程因请求资源而阻塞时，对已获得的资源保持不放。
不剥夺条件：线程已获得的资源，在未使用完之前，不能强行剥夺。
(!!!)循环等待条件：若干线程之间形成一种头尾相接的循环等待资源关系。

通常是线程1持有锁1并等待锁2阻塞，线程2持有锁2并等待锁1阻塞，造成循环等待，导致死锁
常规避免死锁方法：线程加锁顺序一致，都是先获取lock1再获取lock2。

什么是类型擦除

类型擦除是指在编译时，泛型类型的具体类型信息会被擦除，只保留原始类型。也就是说，在运行时，泛型类型的具体类型参数是不可用的。例如，List 和 List 在运行时都会被擦除为 List

为什么要类型擦除

Java 在引入泛型（Java 5）之前，已经存在了大量的非泛型代码。为了确保这些旧代码能够继续运行，Java 需要在不破坏现有代码的基础上引入泛型

related resource: https://article.juejin.cn/post/7474264487513407539

refied

在 Kotlin 中，reified关键字主要用于在泛型函数中实现类型参数的具体化，使得在函数内部可以直接访问类型参数的具体类型，而不仅仅是作为一个占位符。
做到类型擦除后的再生，便于开发者优雅的使用泛型以及获取方法的泛型类型

StateFlow和SharedFlow的区别

StateFlow is a special SharedFlow. with only last one observed when client subcribe on it.
StateFlow holds an value inside (has a value property inside), if you want to update and observe state, you need to use StateFlow.
if you just want to observe event instead of state change, you can use SharedFlow.

Retrofit mechanism

related resource:
https://proandroiddev.com/retrofit-magic-service-interface-8efd486c8996

Retrofit is based on dynamic proxy, user just need to define call service interface.
when we call: retrofit.create(..), it will return a proxy instance, just like we say in the above, so when we call retrofit service method, it will routed to InvocationHandler’s invoke method, and then in this method, we will parse the interface method annotations, and make the http call.

whole process

Validation: Retrofit confirms APIService is valid and annotations are correct.
Dynamic Proxy Creation: Retrofit creates a proxy that intercepts method calls.
Method Blueprint Creation: The getUser() method is parsed into a ServiceMethod.
Runtime Interception: The proxy catches the call and passes it to ServiceMethod.
Request Assembly: Query parameters are encoded, headers added.
HTTP Execution: The request is executed via OkHttp.
Response Conversion: Raw HTTP response is converted to UserResponse using the configured converter.
Suspend Resumption: The coroutine is resumed with the result.
